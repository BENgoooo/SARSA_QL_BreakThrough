🌊 深海迷航 (Deep Sea Dilemma)

基于表格型 SARSA 与 Q-Learning 的极限生存与资源管理挑战

一、 项目背景与初衷

在强化学习中，SARSA (同策略 On-Policy) 与 Q-Learning (异策略 Off-Policy) 的核心差异往往被简化为“悬崖寻路”问题。然而，为了更深刻地理解MDP（马尔可夫决策过程）中的随机性转移以及风险厌恶 vs 期望最大化的本质，我们需要一个更有趣、更贴近真实生存环境的场景。

本项目《深海迷航》将玩家置于一个具有有限资源（氧气）、环境干扰（暗流/打滑）和致命陷阱（海沟）的网格世界中。该场景的状态空间经过精确计算，完美适配表格型（Tabular）算法，无需神经网络即可实现快速收敛，让你将全部精力集中在算法逻辑的对比与调参上。

二、 游戏设定与规则

1. 背景故事

你驾驶一艘微型潜水器，在 6x6 的深海网格区域寻找失落的“亚特兰蒂斯宝藏”。潜水器携带的氧气有限，海底存在不可控的暗流，且部分区域是致命的深海海沟。你的目标是：找到宝藏，并活着返回水面。

2. 核心机制

  • 氧气限制： 初始氧气 15 点，每移动一步消耗 1 点。氧气耗尽则潜水器损毁（Game Over）。
  • 环境打滑（ Slippery Environment）： 在普通水域，潜水器有 10% 的概率不受控制，随机滑向相邻的四个方向之一（模拟水下阻力与操作延迟）。
  • 暗流（Currents）： 特定格子存在暗流。进入暗流格子后，有 80% 的概率被强制推向暗流方向，仅 20% 概率按玩家指令移动。
  • 海沟（Trench）： 地图边缘或特定路径上的深渊。一旦掉入，潜水器被压碎，受到巨额惩罚并结束回合。
  • 补给站： 随机分布的氧气瓶，拾取后氧气 +10。

三、 MDP 建模 (适配表格型算法)

为了保证 Q-Table 能够有效收敛，我们将状态空间进行了严格的离散化控制。

维度 定义 空间大小
位置 (x, y) 网格坐标，6x6 36
氧气 (Oxygen) 离散整数 0 ~ 15 16
宝藏状态 (Has_Treasure) 布尔值 (True/False) 2
总状态数 |S| 36 * 16 * 2 = 1152 个状态
动作空间 |A| 上, 下, 左, 右 (4个离散动作)

Q-Table 规模： 1152 状态 × 4 动作 = 4608 个 Q 值。这个规模对于现代计算机来说，训练几千个 Episode 只需几秒钟，非常适合观察训练过程。

四、 最终目标与核心看点 (算法宿命对决)

本项目的终极目的不是单纯打通游戏，而是观察并验证 SARSA 与 Q-Learning 在面对“高风险捷径”时的不同决策逻辑。

🔍 核心看点：海沟边缘的抉择

假设从起点到宝藏的最短路径必须紧贴着“海沟”边缘行走。由于环境存在 10% 的“打滑”概率：

  • Q-Learning (异策略) 激进/最优主义： 它假设在下一个状态总会采取最优动作。因此，它会忽略打滑掉进海沟的风险，坚定地学习并选择这条贴着海沟的最短路径。在测试时（关闭探索），它能完美走捷径；但在训练时，它经常因为打滑而“作死”。
  • SARSA (同策略) 保守/风险厌恶： 它的更新公式包含了实际的下一个动作（包含 $\epsilon$-greedy 的随机探索）。SARSA 会“意识到”：贴着海沟走，万一打滑或者触发了随机探索，就会掉进海沟拿大负奖励。因此，SARSA 会自动学习出一条远离海沟、稍微绕远但绝对安全的次优路径。

目标验证： 训练结束后，提取两者的 Q-Table 并可视化其策略路线图，你将亲眼看到 Q-Learning 的“贴边捷径”与 SARSA 的“安全绕行”！

五、 算法与代码要求

  • 环境封装： 建议使用 Python 编写一个类似 Gymnasium 接口的环境类 DeepSeaEnv，包含 reset(), step(action), render() 方法。
  • 奖励函数设计 (Reward Shaping)：
    • 每走一步：-1 (促使寻找最短路径，节省氧气)
    • 掉入海沟/氧气耗尽：-100 (回合结束)
    • 拾取氧气瓶：+10，氧气+10
    • 找到宝藏：+50，状态 Has_Treasure 变 True
    • 带宝藏返回水面起点：+200 (回合胜利结束)
  • 算法实现： 分别实现标准的 Q-Learning 更新公式和 SARSA 更新公式。注意处理好 Episode 结束时的边界条件（Terminal State 的 Q 值为 0）。
  • 超参数探索： 尝试调整 $\epsilon$-decay (探索率衰减) 和 $\alpha$ (学习率)，观察它们对“绕路保命”行为的影响。

六、 可视化与评估指标

为了直观感受模型的“探索过程”，要求实现以下可视化模块：

1. 实时探索过程 (基于 Pygame 或 Matplotlib)

  • Pygame 渲染： 编写一个简单的 2D 渲染器，用不同颜色的方块代表潜水器、海沟、暗流和宝藏。在训练时开启慢速渲染，观察 Agent 前期如何像个无头苍蝇一样乱撞、频繁掉入海沟，后期如何学会寻找氧气瓶并规划路线。
  • 轨迹热力图： 记录前 100、500、2000 个 Episode 的 (x,y) 坐标，使用 matplotlib 绘制热力图，直观展示 Agent 探索范围的收缩过程。

2. 策略提取与对比图 (核心成果)

  • 编写一个 plot_policy() 函数。固定 氧气=10，Has_Treasure=False，提取每个网格的最优动作，用箭头 (Quiver Plot) 画在地图上。
  • 将 SARSA 的箭头图与 Q-Learning 的箭头图并排对比，重点观察海沟附近的箭头指向。

3. 评估指标 (Metrics)

  • 收敛曲线： 每回合的累计奖励 (Cumulative Reward) 滑动平均图。
  • 生存率： 每 100 回合中，成功带着宝藏返回水面的比例。
  • 平均步数： 成功通关所需的平均步数（Q-Learning 应该比 SARSA 步数更少，但 SARSA 训练期死亡率可能更高）。

七、 项目实施里程碑

  1. Phase 1: 世界构建 (2天) - 实现 DeepSeaEnv，完成状态映射、打滑逻辑和奖励判定。使用随机动作测试环境是否会正常报错。
  2. Phase 2: 算法注入 (3天) - 编写 Q-Table 初始化、SARSA 和 Q-Learning 的核心循环代码。确保能跑通基本的训练流程。
  3. Phase 3: 可视化接入 (2天) - 引入 Pygame 或 Matplotlib，实现训练过程的动画和轨迹记录。
  4. Phase 4: 对决与分析 (2天) - 固定随机种子，分别训练两个模型。导出 Q-Table，绘制策略箭头对比图，撰写分析报告。
💡 避坑指南： 在离散化状态时，务必将 (x, y, oxygen, has_treasure) 映射为一个唯一的整数 ID（例如使用字典或哈希函数），以便于作为 Numpy 数组（Q-Table）的索引。
