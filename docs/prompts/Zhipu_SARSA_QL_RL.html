Section 01 — 项目背景

为什么是"暗影潜行"？

核心动机

SARSA 与 Q-Learning 的本质区别在于更新公式中对下一动作的选择方式——SARSA 使用实际采取的动作（on-policy），而 Q-Learning 使用状态下的最大 Q 值动作（off-policy）。在静态环境中（如标准迷宫、悬崖寻路），两者的策略差异仅在"悬崖边"这种极端场景才明显。而当环境中存在移动威胁时，这种差异会被系统性地放大：

  • Q-Learning 学到的是"理论上最优路径"——它假设自己不会犯错，因此会贴近守卫走捷径
  • SARSA 学到的是"考虑探索风险后的安全路径"——因为它知道 ε 探索可能让自己撞上守卫，所以主动绕远
DESIGN PRINCIPLE

本项目并非追求游戏花哨，而是让算法本身成为主角。地图、守卫、奖励的每一个参数，都服务于一个目标：让你亲眼看到两个算法在同一个世界中走出不同的路。

Section 02 — 游戏规则

暗影潜行：规则定义

世界设定

一个 8×8 的网格世界，包含以下元素：

  • 墙壁 (W)：不可通行的障碍物，构建迷宫结构
  • 潜入者 (S)：Agent 的起点，位于地图左下区域
  • 宝藏 (G)：目标终点，位于地图右上区域
  • 巡逻守卫 (P)：沿固定路线来回巡逻的敌人，共 2 名

Section 03 — MDP 建模

状态、动作、奖励的精确定义

状态空间 S

状态定义为元组 (agent_r, agent_c, guardA_r, guardA_c, guardB_r, guardB_c)：

# 状态维度计算 Agent 位置: 8×8 = 64 种（去掉墙壁约 40 种可行位置） 守卫 A 位置: 4 种（巡逻路线上的 4 个格子） 守卫 B 位置: 6 种（巡逻路线上的 6 个格子） 总状态数 ≈ 40 × 4 × 6 = 960 个离散状态

约 960 个状态 × 4 个动作 = 3840 个 Q 值，完全可以使用表格法存储，训练收敛快，适合观察算法行为。

KEY INSIGHT

守卫位置的引入是本项目的灵魂——它使得状态空间足够大以展现算法差异，又足够小以确保 Q-table 可行。如果去掉守卫，这就退化为普通迷宫，SARSA 和 Q-Learning 将几乎没有区别。

被捕获后 Agent 回到起点，守卫回到各自巡逻起点，回合继续（不终止，直到最大步数或到达宝藏）。

终止条件
  • 成功终止：Agent 到达宝藏格子
  • 截断终止：单回合超过 max_steps=200 步
  • 被守卫捕获不终止，而是重置到起点继续当前回合（这样 Agent 必须学会"活着比冒险更重要"）
Section 04 — 算法要求

SARSA 与 Q-Learning 的实现与对比

SARSA (On-Policy)

使用实际执行的下一动作 a' 来更新 Q 值。因为 ε-greedy 探索可能选到危险动作，所以 Q 值中自然包含了对探索风险的评估，最终学到更安全但更长的路径。

Q(s,a) ← Q(s,a) + α[ r + γ·Q(s',a') - Q(s,a) ] /* a' 是 ε-greedy 实际选的 */

Q-Learning (Off-Policy)

使用下一状态的最大 Q 值来更新，不考虑探索。它假设自己永远选最优动作，因此更倾向理论最短路径，即便那条路紧贴守卫。

Q(s,a) ← Q(s,a) + α[ r + γ·max Q(s',·) - Q(s,a) ] /* 直接取最大，忽略探索 */
实现要求
  • 两个算法共享相同的环境、超参数、随机种子，仅更新公式不同
  • 使用 defaultdict 或 numpy array 存储 Q-table，key 为 (state, action)
  • ε-greedy 策略中，ε 从 1.0 线性衰减到 0.05（前 70% 训练轮次衰减，后 30% 保持 0.05）
  • 每个算法训练 5000 个 episode，记录每轮的累计奖励、步数、是否成功、被捕获次数
推荐超参数
超参数推荐值说明
α (learning_rate)0.1学习率，不宜过大以保持稳定
γ (discount_factor)0.95折扣因子，较重视长期回报

Section 05 — 可视化要求

让训练过程"看得见"

可视化一：实时网格动画

训练过程中，每隔 N 个 episode 回放一次该轮的完整轨迹：

  • Agent 用绿色圆点表示，守卫用红色圆点表示
  • Agent 走过的格子留下渐变足迹（步数越大颜色越淡）
  • 被捕获时显示闪烁红色，到达宝藏时显示闪烁金色
  • 支持暂停/继续/调速（0.5x ~ 5x）
  • 推荐使用 matplotlib.animation 或 pygame 实现
可视化二：训练曲线对比

在同一张图中绘制 SARSA 和 Q-Learning 的以下指标（滑动平均窗口=50）：

  • 累计奖励曲线：最核心的对比——SARSA 的平均奖励是否更稳定？Q-Learning 是否波动更大？
  • 成功率曲线：到达宝藏的回合占比
  • 被捕获率曲线：每轮被捕获的次数——这是两者差异最直观的体现
  • 平均步数曲线：SARSA 是否走了更多步？
可视化三：Q-Value 热力图

训练结束后，固定守卫在某个位置，对每个 Agent 可达格子计算 max Q(s,·)，绘制热力图：

  • 对比 SARSA 和 Q-Learning 的价值分布差异
  • 观察靠近守卫巡逻路线的格子：SARSA 是否赋予了更低的 Q 值？
  • 在热力图上叠加最终策略的箭头图（每个格子画 argmax 方向箭头）
可视化四：最终策略轨迹叠加

训练完成后，用 ε=0（纯贪心）分别运行 10 次，将所有轨迹叠加绘制：

  • SARSA 的轨迹应明显绕开守卫巡逻区域
  • Q-Learning 的轨迹应更贴近守卫走最短路径
  • 用不同颜色区分两个算法的轨迹，叠加在同一张地图上
Section 06 — 评估指标

如何判断项目成功

核心验收指标
指标预期现象反映的算法本质
训练后期被捕获率 SARSA < Q-Learning On-policy 考虑探索风险，学会了避险
训练后期成功率 两者接近（均 > 90%） 两个算法都能收敛到有效策略
训练后期平均步数 SARSA > Q-Learning SARSA 绕远路换安全
训练期累计奖励方差 SARSA 更小 On-policy 策略更稳定保守

SUCCESS CRITERIA

项目完成的核心标志：你能够不查资料，用自己的话解释为什么在同一个环境中，SARSA 学到了绕远路而 Q-Learning 走了险路，并能在可视化结果中指出具体证据。

Section 07 — 实施路线

里程碑计划

1
环境搭建
实现 ShadowEnv 类：网格初始化、守卫巡逻逻辑、step 函数、reset 函数、渲染函数。手动测试：随机动作跑 100 步，确认守卫正确巡逻、碰撞正确判定、奖励正确返回。
2
Q-Learning 实现
实现 Q-Learning Agent（Q-table、ε-greedy、update 函数）。在环境中训练 5000 轮，绘制累计奖励曲线确认收敛。
3
SARSA 实现
实现 SARSA Agent（注意 a' 的选择时机——先选 a'，再用 a' 更新，然后将 a' 作为下一步动作执行）。同样训练 5000 轮。
4
可视化系统
实现四种可视化：网格动画回放、训练曲线对比、Q 值热力图 + 策略箭头、最终轨迹叠加。建议使用 matplotlib，网格动画可用 matplotlib.animation.FuncAnimation。
5
对比分析报告
撰写 1-2 页的分析文档，用可视化结果作为证据，回答：两个算法的策略有何不同？为什么？调节 ε 或守卫速度后差异如何变化？
Section 08 — 进阶实验

完成基础目标后的拓展方向

实验一：守卫速度的影响

将守卫改为"每 2 步移动 1 次"或"每步移动 2 格"，观察 SARSA 与 Q-Learning 差异的变化趋势。思考：当守卫越慢，两者差异越小还是越大？为什么？

实验二：地图尺寸扩展

将地图扩展到 10×10 或 12×12，增加更多墙壁和守卫。观察 Q-table 的收敛速度如何变化，思考：表格法的瓶颈在哪里？这为后续 DQN 的学习埋下伏笔。

实验三：ε 对策略差异的影响

分别用 ε=0.01, 0.1, 0.3 训练两个算法。当 ε 很小时，SARSA 和 Q-Learning 的策略是否趋于一致？这揭示了 on-policy 和 off-policy 差异的根源是什么？
