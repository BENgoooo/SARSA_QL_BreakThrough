Section 01

项目背景与动机

强化学习的核心概念——状态、动作、奖励、策略、价值函数——需要通过动手实践才能真正内化。SARSA（on-policy）和 Q-Learning（off-policy）是课程中介绍的两种最基础的时序差分（TD）控制算法，它们的关键差异在于：SARSA 学习的是"当前策略下"的价值，而 Q-Learning 学习的是"最优策略下"的价值。

然而，许多课程作业（如悬崖寻路）中两者的性能差异不够直观，难以激发深入思考。本项目选择贪吃蛇（Snake）作为实验环境，其游戏规则简单但策略空间足够丰富，能够清晰地展现出两种算法在安全性与最优性之间的权衡差异。

核心目标：不是做出一个"花哨的游戏 AI"，而是通过一个足够合适且有趣的场景，让你亲眼看到 SARSA 和 Q-Learning 在相同环境下做出不同决策的过程——这才是真正的算法理解。

Section 02

为什么选择贪吃蛇

✓ 教学适配度高
  • 状态空间可控：经合理离散化后，状态数约 300 个，Q 表完全可管理
  • 奖励结构丰富：吃食物（正奖励）、死亡（负奖励）、存活（小负/零奖励）形成多层级奖励信号
  • 策略差异显性化：蛇身附近的狭窄通道中，SARSA 倾向保守绕行，Q-Learning 可能选择冒险抄近路——差异一目了然
  • 无需深度学习：经典 Q 表方法即可训练出有效策略
★ 趣味性与交互性
  • 直观可视：蛇的移动轨迹、探索路径可直接渲染在网格上
  • 渐进成就感：从随机乱走到流畅吃食，训练效果肉眼可见
  • 对比实验天然：同一环境两组 Q 表，分屏/切换对比，差异清晰
  • 可扩展性：后续可添加障碍物、多蛇对战等高级玩法
Section 03

游戏规则与环境设计

▦ 基础规则

在经典贪吃蛇规则基础上做简化，使其适合 Q 表学习：

  • 地图为 10×10 的网格，四周有墙壁（碰撞即死亡）
  • 蛇初始长度为 3 节，位于地图中央附近
  • 每个时间步，蛇向某个方向移动一格
  • 吃到食物后蛇身增长 1 节，食物随机刷新到空位
  • 死亡条件：撞墙 或 撞到自身
  • 单局最大步数限制为 200 步（防止无限循环）

Section 04

状态空间与动作空间定义

⚙ 动作空间

动作空间 A = {上, 下, 左, 右}，共 4 个离散动作。不允许蛇直接掉头（即不能从"向右"直接变为"向左"），实现时需过滤无效动作。

⚙ 状态空间（关键设计）

为了让 Q 表可控（避免维度爆炸），不直接使用坐标作为状态，而是将每个状态编码为局部感知特征的组合：

危险感知 正前方、左方、右方是否有障碍（墙/自身）
2^3 = 8 种
移动方向 当前蛇的移动方向（上/下/左/右）
4 种
食物方向 食物相对蛇头的方位（8方向 或 简化为4方向）
4 或 8 种
可达性 （可选进阶）蛇头附近通道是否狭窄
2 种（宽/窄）

最小状态空间：8 × 4 × 4 = 128 个状态

推荐状态空间：8 × 4 × 8 = 256 个状态

每个状态只需查询 Q 表中的 4 个动作值，Q 表大小约 256 × 4 = 1024 个条目——训练极快，便于实时可视化。

为什么这样设计状态？

直接使用 (x, y, 蛇身形状) 会导致状态空间爆炸（指数级）。而局部特征编码让智能体学会"通用规则"（如：前方有障碍就转向，食物在左边就左转），而不是记忆每一步的位置——这正是 SARSA/Q-Learning 这类表格型方法的正确使用方式。

Section 05

奖励函数设计

奖励函数是引导算法行为的核心，需精心设计：

事件 奖励值 设计理由
吃到食物 +10 主要正向激励，鼓励蛇主动寻找食物
死亡（撞墙/撞自身） -10 主要惩罚，让智能体学会避开危险
每步存活 -0.1 微小惩罚，防止蛇原地打转/拖延，鼓励尽快吃到食物
靠近食物（距离减少） +1 （可选）中间引导信号，帮助早期学习。SARSA 对此更敏感
远离食物（距离增加） -1 （可选）与"靠近食物"配合使用

可选奖励消融实验：分别在"有/无中间引导信号"下训练 SARSA 和 Q-Learning，观察信号对两者探索行为的不同影响——SARSA 因 on-policy 特性会更保守地跟随引导，而 Q-Learning 可能忽略引导直接探索更短路径。

Section 06

算法要求：SARSA 与 Q-Learning

∞ SARSA（on-policy TD control）
Q(S, A) ← Q(S, A) + α [R + γ · Q(S', A') - Q(S, A)]
// 注意：A' 是用当前策略（如 ε-greedy）实际选择的下一个动作

实现要求：

  • 使用 ε-greedy 策略选择动作（探索率 ε 从 1.0 线性衰减至 0.01）
  • 学习率 α = 0.1，折扣因子 γ = 0.95
  • 每步更新：先执行 A，观察 R 和 S'，用当前策略选择 A'，然后更新 Q(S,A)
⚡ Q-Learning（off-policy TD control）
Q(S, A) ← Q(S, A) + α [R + γ · max_a Q(S', a) - Q(S, A)]
// 注意：取 S' 下所有动作的最大 Q 值，与实际执行的动作无关

实现要求：

  • 使用 ε-greedy 策略选择动作（同 SARSA，相同的 ε 衰减方案）
  • 学习率 α = 0.1，折扣因子 γ = 0.95
  • 每步更新：先执行 A，观察 R 和 S'，直接取 max_a Q(S',a) 更新
🔧 通用实现要求
  • Q 表初始化：全部初始化为 0
  • ε 衰减策略：ε = max(0.01, 1.0 - episode / 500)，在 500 个 episode 内从 1.0 线性衰减到 0.01
  • 训练轮数：至少 2000 个 episode
  • 每 episode 最大步数：200 步
  • 代码结构：SARSA 和 Q-Learning 共享同一个环境类，仅更新规则不同——方便对比
  • 编程语言：Python 3，推荐使用 NumPy 进行 Q 表操作
  • 渲染库：推荐 Pygame 或 Matplotlib 均可
Section 07

SARSA vs Q-Learning 在贪吃蛇中的预期差异

这是本项目的核心探究问题。以下是你应该在实验中观察到的行为差异：

维度 SARSA（on-policy） Q-Learning（off-policy）
策略风格 偏保守。Q 值反映的是 ε-greedy 策略下的价值（包含随机探索的"风险"），因此学到的路径会远离危险区域 偏激进。Q 值反映的是最优策略下的价值，学到的路径更短但可能紧贴墙壁或蛇身
狭窄通道 倾向于绕行，因为狭窄通道中 ε-greedy 探索有概率撞墙，其 Q 值会因"探索风险"被打压 倾向于穿越，因为 max 操作只看最优路径，忽略探索噪声
死亡率 训练后期死亡率更低（更安全的策略） 训练后期可能仍有一定死亡率（最优路径本身有风险）
平均得分 可能略低——安全但不够高效 可能更高——激进但高效
收敛速度 相对较慢（因为 Q 值估计更"诚实"，波动更小但进步也更慢） 相对较快（因为 max 操作让价值信号传播更快）
类比 像一个"谨慎的新手司机"——宁可多绕路也不冒险超车 像一个"自信的老司机"——走最短路径，但偶尔会出事故

实验验证建议：在训练过程中选取同一个测试 episode（固定 ε=0，不探索），分别用两个模型运行，逐帧记录蛇的移动轨迹。将两条轨迹叠加在同一张地图上对比——你将直观看到 SARSA 走出的"安全弯路"和 Q-Learning 走出的"激进捷径"。

Section 08

训练过程可视化要求

可视化是理解算法行为的关键。要求实现以下三层可视化：

🎬 Level 1：实时游戏画面
  • 在网格地图上实时渲染蛇身、食物、墙壁
  • 用不同颜色标注蛇头方向、Q 值最大的动作方向
  • 可调节渲染速度（慢速观察 / 快速训练）
  • 显示当前 episode、步数、得分、ε 值
📊 Level 2：训练曲线
  • 实时绘制 每个 episode 的得分（折线图 + 滑动平均）
  • 实时绘制 每个 episode 的存活步数
  • SARSA 与 Q-Learning 的曲线叠加在同一张图上，用不同颜色区分
  • 推荐使用 Matplotlib 动态更新（plt.ion()）或 Pygame 内嵌绘图
🔎 Level 3：Q 值热力图（进阶）
  • 对于地图上每个格子，用颜色深浅表示该位置的最大 Q 值
  • 用箭头表示每个位置 Q 值最大的动作方向
  • 每隔 N 个 episode 更新一次，形成"Q 值逐渐成型"的动态过程
  • SARSA 和 Q-Learning 的热力图并排显示

分屏对比方案：建议将游戏窗口分为左右两半，左边 SARSA，右边 Q-Learning，同步运行同一个初始状态的 episode——你可以实时看到两条蛇在同一个地图上走出不同路线。
